# GM（Growth Mentoring）マニュアル

評価シートを自分で動かせるようにする育成プログラム。
Claude Codeがナビゲーターとして評価シートの読み解きとアクションプラン策定を手伝います。

---

## 0. 事前準備

### Claude Codeのインストール

GMはClaude Code（Anthropic公式のCLIツール）上で動きます。

1. Claude Codeをインストール

**Mac / Linux:**
```bash
curl -fsSL https://claude.ai/install.sh | sh
```

**Windows:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

2. 初回起動して認証

```bash
claude
```

ブラウザが開くので、Anthropicアカウントでログインしてください。
※ Anthropicアカウントがない場合は https://console.anthropic.com/ で作成。

### Gitの確認

```bash
git --version
```

バージョンが表示されればOK。表示されない場合は https://git-scm.com/ からインストールしてください。

## 1. セットアップ（初回のみ・10分）

### Step 1: リポジトリをクローンする

ターミナル（WindowsならGit Bash、Macならターミナル）を開いて、以下を実行：

```bash
git clone https://github.com/Kozue-front/mentor-board-template.git
```

`mentor-board-template` というフォルダが作成されます。

```bash
cd mentor-board-template
```

> **「git cloneって何？」という方へ**
> GitHubというサービスからファイル一式を自分のPCにコピーするコマンドです。
> コピーされたフォルダの中にGMの仕組みがすべて入っています。

### Step 2: 評価シートCSVを置く

1. Googleスプレッドシートの自分の評価シートを開く
2. メニュー → `ファイル` → `ダウンロード` → `カンマ区切り形式(.csv)`
3. ダウンロードしたCSVファイルを `evaluation/` フォルダに入れる

```
mentor-board-template/
└── evaluation/
    └── 会社名_あなたの名前_評価 - 2026年3月.csv  ← ここに置く
```

> **CSVがなくても始められます。**
> Claude Codeに口頭で評価内容を伝えればOKです。ただしCSVがあるほうが正確です。

### Step 3: プロフィールを書く

`profile.md` をテキストエディタ（VSCode、メモ帳など）で開いて、自分の情報を埋めてください。

**これだけは入れてほしい（3項目）:**

| 項目 | 書く内容 | 例 |
|------|---------|-----|
| 職務経歴 | 直近2〜3社の経歴・役割 | 「○○社でWebディレクター3年」 |
| スキル・ツール | 使えるツール、得意領域 | 「GA4, HubSpot, Figma」 |
| ポジショニング | 自分が目指す方向性（一言） | 「データに強いマーケター」 |

**あると精度が上がるもの（推奨）:**

| 項目 | 書く内容 | どう使われるか |
|------|---------|---------------|
| MBTI | 16タイプの結果 | フィードバックのトーン調整に使う |
| クリフトンストレングス | TOP5〜全34資質 | 強みを活かしたアクションプラン設計 |
| 収入・目標 | 現状と目指すライン | キャリア設計の相談で使う |
| 本人の志向 | 好きな仕事、避けたいこと | 「この人に合う」提案をするため |

> **参考**: プロフィールにMBTI、クリフトンストレングス全34資質、職務経歴書をフルで入れているメンバーもいます。
> 情報が多いほどメンターのフィードバックが具体的になります。書ける範囲で大丈夫です。

## 2. GMを起動する

### Claude Codeを開く

ターミナルでリポジトリのフォルダに移動して、Claude Codeを起動：

```bash
cd mentor-board-template
claude
```

Claude Codeが起動したら、以下のように話しかけてください：

```
評価シートを見てほしい
```

### 初回チュートリアル

まず進捗バーが表示されます：

```
═══ Status ═══
Lv.1 | XP: 0/150
░░░░░░░░░░░░░░░ 0%
```
※ ゲーミフィケーション設定済みの場合のみ。詳細は `gm/gamification.md` を参照。

続けて初回アナウンスが出ます：

```
「はじめまして。
 評価シートを使った育成プログラムの案内をしますね。

 評価シートの読み方と使い方を伝えるのが私の役割です。
 最終的な評価判断は評価者（人間）が行うので、
 私の役割は構造の整理です。

 では、評価シートについてお伺いします。
 まず最初にどちらをやりたいですか？

 1. 評価シートを書くのを手伝ってほしい
 2. 評価シートはあるので、メンターと1on1やFBがもらえるようにしたい

 ※ 書くのを手伝う場合は、設問に答えてもらって
   書ききってからメンター作成に進みます。
   すぐにメンターを作成したい場合は2を選んでください。」
```

#### ルートA: 評価シートを書くのを手伝ってほしい

以下のフェーズを順に進みます。途中で「いったんここまでにする」と言えば中断でき、次回の続きから再開できます。

1. **スコープ確認** — どこまで一緒にやるか選択（ゴールだけ / ゴール+マイミッション / 全部）
2. **評価シート読み込み** — CSVがあれば読み込み、6軸の現在レベルと評価者コメントを提示
3. **ゴール設定** — 前期のゴール・テーマがあれば先に提示し「前期はこうでした。今期はどうしますか？」の流れで設定。継続/変更いずれの場合も理由を一言聞く。未達継続の場合、原因分析はメンターに委ねる
4. **テーマ設定（1〜3つ）** — ゴールに紐づくテーマ候補を提示。各テーマごとに「これでいく / 調整したい / スキップ」で選択
5. **各軸のマイミッション設定**（スコープ2以上） — 6軸を順番に回し、現在レベルと次レベルの定義を提示。本人の回答を評価シートの言語に翻訳して候補提示。「これでいく / 調整したい / 既存のままでいく / スキップ」で選択
6. **アクションプラン設定**（スコープ3のみ） — 実案件・実業務に絞った候補を動的に生成。各アクションに「成果物（何を作るか）」と「提出先（誰に見せるか）」を含める
7. **整合性チェック（自動）** — ゴール・テーマ・マイミッションの矛盾がないかチェック。矛盾があれば指摘し、揃えるかどうかを選択
8. **構造分析FB** — 評価シートに対して構造的なFBを返す。その後メンター作成（共通Phase C）へ誘導

#### ルートB: 評価シートはあるのでFBが欲しい

1. **評価シート読み込み** — CSVを読み込み、6軸の現在レベルと評価者コメントを提示
2. **FBデモ** — 重みが大きい軸でスコアが低いところの指摘、コメントとマイミッションの整合性チェック、伸びしろのポイント提示
3. メンター作成（共通Phase C）へ

#### 共通Phase C: メンター作成

評価シートの設定が完了したら、メンター作成に進みます。

1. **スキル抽出と推薦** — 評価シートからスキル領域を自動抽出し一覧で提示。3つおすすめを推薦。ユーザーが番号で選択（「おすすめで」もOK）。選択後に「この3つのスキルに力を入れたい理由を一言もらえますか？」と聞く
2. **メンター割り当て提示** — 選択されたスキルに対応するメンター候補をテーブルで提示
3. **メンターFBデモ（必須）** — メンターのペルソナで評価シートに対してFBを返す。体験させる前提で進行。ここがユーザーにとっての「ゲーム開始」の瞬間
4. **メンターチューニング** — FBの後、以下の調整質問を行う（回答が実際のシステム調整に直結する）:
   - **口調**: ちょうどいい / きつい→柔らかくしてほしい / 柔らかすぎる→もっとストレートでいい
   - **内容**: 気づきがあった→このメンターでいく / 的外れだった→他のメンターを試したい / 他を試したいが今日は時間がない（後日再設定できます）
   - 口調の調整が必要 → ペルソナの語り口パラメータを実際に変更
   - 人選の変更が必要 → 別のメンター候補を提示して再選択

#### チュートリアル完了後の案内

セットアップ完了後、次回以降にできることと推奨頻度の案内が出ます:

- 日報を書く → ログが溜まる → 月末に自動集計
- メンターに壁打ち → アクションプランの質を上げる
- 「評価シートを見て」→ 構造分析・整合性チェック
- 「振り返りをまとめて」→ 四半期末の整形出力

次回の利用タイミングも聞かれます（毎日 / 週1 / 月1 / 必要なときだけ）。カレンダー登録は自分で行います。

チュートリアル完了時にXPは入りません。アクションを実行して成果物が出たときに入ります。

あとは質問に答えていくだけです。

## 3. 各メニューの詳細

### メニュー1: 評価シートを書くのを手伝ってほしい

**いつ使う？** → 評価シートの単月振り返り（KEEP/PROBLEM/TRY）がまだ空のとき

**流れ:**
1. 「今月やったことを教えてください」と聞かれる
2. やったことを箇条書きで入力（ざっくりでOK）
3. 「他にありますか？」→ あれば追加、なければ「ないよ」
4. KEEP/PROBLEM/TRYに整形された振り返りが出てくる
5. 内容を確認して「OK」で評価シートCSVに反映

**入力例:**
```
LinkedInでの発信、自社マーケのKPI設定、クライアントのCVR改善施策
```

### メニュー2: メンターとFBがもらえるようにしたい

**いつ使う？** → 初回セットアップ時。評価シートの課題にメンターをマッチングする

**流れ:**
1. 評価シートの全軸を分析して、課題を抽出
2. 各課題に最適なメンター（1〜2名）をマッチング
3. なぜそのメンターなのかの理由が提示される
4. マッチング結果の調整（入れ替え・追加）ができる

**出力例:**
```
課題1: データレイヤー習得 → Avinash Kaushik × Scott Brinker
課題2: CVR改善のドキュメント化 → Peep Laja × Andy Grove
```

> **メンターとは？**
> 実在の著名人（HubSpot共同創業者のDharmesh Shah、Intelの元CEOのAndy Groveなど）の
> 著書・講演・思考フレームワークをベースにした、AI壁打ち相手です。
> 本人が直接指導するわけではありません。
> メンター一覧は `mentor/mentor-board.md` を参照。

### メニュー3: メンターに壁打ちしたい

**いつ使う？** → 具体的な課題や悩みについて、特定のメンターに相談したいとき

**前提:** メニュー2でメンターマッチングが済んでいること

**流れ:**
1. 評価シートから見える課題一覧とおすすめメンターが表示される
2. 課題番号を選ぶか、自由に「○○について△△に聞きたい」と入力
3. メンターのペルソナに基づいたフィードバックが返ってくる
4. 深掘りしたければ続けて質問できる
5. 「最初の画面に戻る」でメニューに戻る

**入力例:**
```
セキュリティ×HubSpotのポジション確立についてBruce SchneierとDharmeshに聞きたい
```

### メニュー4: 日報を書く

**いつ使う？** → 毎日（推奨）。最低でも月1回

```
日報
```

と打つだけ。3つの入力方法があります:
1. テキストで書く（3項目＋お気持ち・1〜3分）
2. スクショを貼る（作業画面やAsanaのスクショ→自動で日報化）
3. コピペ（タスク一覧等を貼る→自動で整形）

#### テキスト入力（簡単入力）

3つだけ聞かれます:
- **今日やった仕事**（何でもOK）
- **今日の「できた」か「困った」どちらか一つ**
- **誰かとやりとりしたこと**（あれば。なければスキップ）
- **＋ お気持ち**（任意。一言でも愚痴でもOK）

所要時間: 1〜3分。

#### テキスト入力（詳細入力）

`日報 詳細` と入力すると、簡単入力の3項目＋お気持ちに加えて以下を追加:
- **明日やること**（なければスキップでOK）

所要時間: 3〜5分。

#### 日報の出力

記録完了後、以下が表示されます:
- やったこと → **関連する評価軸の自動マッピング**
- できた/困った → 「困った」の場合、関連メンターの推薦
- やりとり → 関連軸（クライアント対応力 or チーム貢献）
- お気持ち
- **今月の累積プログレスバー**（軸ごとの件数）

```
今月の累積（3/20時点）
専門性: ████░░ 12件 | 実行力: ██░░░░ 5件 | 対応力: █░░░░░ 3件
```

#### メンターのねぎらい（日報記録完了後・毎回）

日報の記録完了後、設定済みメンターからランダムで1人が出現し、日報の内容を読んだ上で一言コメントします。

- 毎回出現（「今日は誰だろう」のワクワクを作る）
- 設定済みメンターからランダム選出
- **日報の内容に触れたコメント**にする（定型文のローテーション禁止）
- 1〜2行で短く。長いFBではなくねぎらい
- 「困った」が入っていた場合、そのスキルに関連するメンターが出やすくなる重み付けあり

保存後、進捗バー（XP獲得・レベル進捗）が表示され、最初の選択画面に戻れます。

**入力例:**
```
クライアントAのGA4レポート作成、社内勉強会の資料準備、LinkedIn投稿1本
```

**メンターFBの例:**
```
Andy Grove からの一言：
3つとも「仕組みを作る」仕事だ。高レバレッジ活動に分類できる。
ただし問いを1つ残しておく。「今日作った仕組みのアウトプット指標は何だ？」
```

### メニュー5: 伸ばすべきスキルを整理したい

**いつ使う？** → 自分の成長の方向性を確認したいとき

**流れ:**
1. 評価シートと直近の日報を照合
2. 優先度別（最優先/高優先/中優先/継続注意）にスキルが一覧化される
3. 各スキルに「現状」「目指す状態」「効く評価軸」が表示される
4. 深掘りしたいスキルがあればメンターに壁打ちできる
5. `growth/` に保存できる

## 4. 定期的にやること

| タイミング | やること | コマンド |
|-----------|---------|---------|
| **毎日（推奨）** | 日報を書く | メニュー4 |
| **週1（推奨）** | メンターに壁打ち | メニュー3 |
| **月末** | 月次振り返り | 「月次振り返り」と入力 |
| **四半期末** | 振り返り整形 | 「振り返りをまとめて」と入力 |

### 月次振り返り

```
月次振り返り
```

と入力するだけ。日報データから自動集計し、以下を生成します:

- **今月の活動サマリー** — 実施アクション件数、関連評価軸の内訳（プログレスバー付き）
- **スキル推移** — 月初との比較、Lv条件への接近度
- **単月振り返り下書き（評価シート用）** — 得られたこと、KEEP / PROBLEM / TRY を日報から自動要約

生成後、「このまま転記する / 修正してから転記する / 転記せず保存だけする」を選べます。

月次振り返りを「月末に頑張って書く」のではなく、「日報が溜まって自動生成される」構造です。

### 四半期末の振り返り整形

```
振り返りをまとめて
```

3ヶ月分の振り返りを評価シートのフォーマットに整形します。評価者に提出する形で出力されます。

## 5. 復帰セッション（途中離脱・再開時）

しばらく空けて戻ってきた場合、「おかえりなさい」で始まります（「お久しぶり」「しばらくぶり」は使いません）。

前回の状態テーブルが表示されます:

| 項目 | 状態 |
|---|---|
| ゴール・テーマ | 設定済み/未設定 |
| マイミッション | 設定済み/未設定 |
| アクションプラン | 設定済み/未設定 |
| メンター | 名前（設定済み）/未設定 |
| メンターの調整 | 完了/途中 |

続けて再開の選択肢が出ます（状態に応じて不要な選択肢は非表示）:
1. 前回の続きから再開
2. マイミッションの設定に進む
3. メンターに壁打ちしたい
4. メンターを入れ替えたい
5. 新しい評価シートを読み込む（評価期間が変わった場合など）

放置ペナルティなし。放置期間に関する言及もしません。

## 6. 設計原則: AIによる思考停止の防止

このシステムはAIが評価シートの構造整理を手伝うが、AIに選んでもらうだけで本人が考えなくなるリスクがある。以下のルールで防止する。

- **選択式の後に必ず理由を一言添えさせる**: 「番号を選ぶだけ」で進めない。「なぜそれを選びました？」を挟む。ただし誘導的な例文は出さない（コピペされるため）
- **メンターFBの後はチューニング質問を挟む**: 漠然とした感想は聞かない。回答がシステム調整に直結する具体的な質問をする（口調・人選の調整）
- **「答え」を出さない。「構造」を出す**: 最終的に「これでいく」と決めるのは必ず本人
- **「既存のまま」「スキップ」を正当な選択肢として常に提示する**: AIに流されて全部書き換えてしまうリスクを防ぐ

## 7. トラブルシューティング

### 「claude」コマンドが見つからない

インストールコマンドを再実行してください：
- Mac/Linux: `curl -fsSL https://claude.ai/install.sh | sh`
- Windows: `irm https://claude.ai/install.ps1 | iex`

ターミナルを再起動すると反映される場合もあります。

### 「git clone」でエラーが出る

「Permission denied」や「Repository not found」が出る場合：
- GitHubアカウントにログインしているか確認
- リポジトリのCollaboratorに追加されているか管理者に確認してください

### 評価シートCSVが読み込めない

- CSVの文字コードがUTF-8か確認（GoogleスプレッドシートからダウンロードしたものならUTF-8です）
- ファイル名に日本語が入っていてもOKです
- CSVがなくても、口頭で評価内容を伝えれば動きます

### Claude Codeの画面が固まった

`Ctrl + C` で中断できます。もう一度 `claude` で起動し直してください。
途中のデータ（日報・振り返り等）は `sessions/` に保存済みなので失われません。

### 日報が保存されない

`sessions/daily/` フォルダが存在するか確認してください。

### メンターが反応しない

メンターのペルソナファイルが `mentor/personas/` に存在するか確認してください。未設定の場合はチュートリアル（選択肢1か2）を先に完了してください。

### 進捗バーが表示されない

ゲーミフィケーション設定が必要です。`gm/gamification.md` を参照してください。

## 8. よくある質問

**Q: 毎日やらないとダメ？**
A: 最低月1回開いてもらえると、月次振り返りのデータが溜まります。毎日やれば日報ログが充実しますが、強制ではないです。

**Q: AIの判断は評価に反映される？**
A: AIは構造の整理だけです。最終的な評価判断は評価者（人間）が行います。

**Q: メンターって誰？**
A: 実在の著名人（経営者・マーケター等）の著書やフレームワークをベースにしたAI壁打ち相手です。本人が直接指導しているわけではありません。チュートリアル（メニュー2）の中で、あなたの評価シートに合ったメンターを一緒に設定します。

**Q: profile.mdにどこまで書けばいい？**
A: 最低限は職務経歴・スキル・ポジショニングの3つ。MBTIやクリフトンストレングスがあるとメンターのFB精度が大きく上がります。書ける範囲で大丈夫です。

**Q: Cursorでも使える？**
A: Claude Code以外のMCP対応ツール（Cursor等）でも使えますが、動作検証はClaude Codeのみで行っています。

**Q: 途中でやめても大丈夫？**
A: いつでも中断・再開できます。戻ってきたときは「おかえりなさい」で前回の状態テーブルが表示され、続きから始められます。日報や振り返りは `sessions/` に保存されているので、データは失われません。

**Q: 他の人に自分の評価シートが見られる？**
A: 見られません。評価シートCSVは `.gitignore` で除外されているため、GitHubにアップロードされることはありません。すべてあなたのPC内にのみ保存されます。

## 補足: v24ナビゲーターをMCPで接続する場合

> **注意**: v24ナビゲーターはMCPサーバー経由で動作するため、レスポンスが重くなります。
> 通常はローカルのメンターボードで十分ですが、ナビゲーター機能（評価シートの構造分析・整合性チェック等）を強化したい場合に接続してください。

### v24ナビゲーターの接続方法

```bash
# リポジトリ取得
git clone <v24リポジトリURL（管理者から共有されます）>
cd v24
npm install

# Claude Codeに登録
claude mcp add -s user v24 -- node /path/to/v24/v24-mcp-server.js
```

※ `/path/to/v24/` は自分のv24フォルダの絶対パスに置き換えてください。
※ 登録後、Claude Codeを再起動してください。

確認:
```bash
claude mcp list
```
v24が表示されればOK。

### v24の削除方法

```bash
claude mcp remove v24
```

ペルソナファイルやログはローカルに残りますが、v24が起動しなくなるだけです。
