# GM クイックスタートガイド（Cursor版）

評価シートを自分で動かせるようにする育成プログラム。
CursorのAIエージェントがナビゲーターとして評価シートの読み解きとアクションプラン策定を手伝います。

---

## 始め方（3ステップ）

### Step 1: リポジトリをクローンする

```bash
git clone https://github.com/Kozue-front/mentor-board-template.git
cd mentor-board-template
```

### Step 2: Cursorでフォルダを開く

```
Cursor → File → Open Folder → mentor-board-template を選択
```

### Step 3: 評価シートを用意する

自分の評価シートCSVを `evaluation/` フォルダに置く。
なければ口頭で伝えてもOK。

これだけで始められます。npm installやMCP接続は不要です。

---

## Cursorの設定確認

### AIモデルの確認
- Cursor Settings → Models → **Claude系モデル（Claude Sonnet 4以上推奨）** が有効になっていること
- GPT系モデルでも動作しますが、メンターの口調再現精度はClaude系が高いです

### CLAUDE.mdの自動読み込み
- プロジェクトルートに `CLAUDE.md` があれば、CursorのAgent/Composer機能が自動で読み込みます
- 特別な設定は不要です

---

## 使い方

### Composer（Agent mode）を使う

Cursorの右側パネルで **Composer** を開き、モードを **Agent** に切り替えます。

チャット欄に以下のように入力してください:

```
評価シートを見てほしい
```

`CLAUDE.md` のルールに従って、GMフローが起動します。

---

### 初回: チュートリアル

進捗バーが表示されます:
```
═══ Status ═══
CMO Lv.2 マーケ設計者 | XP: 210/400
▓▓▓▓▓▓▓▓▓▓░░░░░ 53%
経営思考 Lv.2 | 意思決定 Lv.1 | 外部発信 Lv.2 | コミュニケーション Lv.2
```
※ ゲーミフィケーション設定済みの場合のみ。詳細は `gm/gamification.md` を参照。

続けて2つの選択肢が出ます:

1. **評価シートを書くのを手伝ってほしい** → 設問に答えていくと評価シートが書ける
2. **評価シートはあるので、メンターとFBがもらえるようにしたい** → すぐにメンター設定へ

※ 日報・壁打ち・スキル整理は、評価シートとメンターの設定が終わってから使えるようになります

---

### 日常: 日報

Composerで以下のように入力:

```
日報
```

3つだけ聞かれます:
- **今日やった仕事**（何でもOK）
- **今日の「できた」か「困った」どちらか一つ**
- **誰かとやりとりしたこと**（あれば。なければスキップ）
- **＋ お気持ち**（任意。一言でも愚痴でもOK）

所要時間: 1〜3分。

記録完了後、設定済みメンターからランダムで1人がねぎらいコメントをくれます。

---

### メンターに壁打ち

```
Dharmeshに壁打ちしたい
```

設定済みのメンターに、案件やアクションプランの相談ができます。
メンター一覧は `mentor/mentor-board.md` を参照。

---

### 月末: 月次振り返り

```
月次振り返り
```

日報データから自動集計し、評価シート用の振り返り下書きを生成します。

---

### 四半期末: 振り返り整形

```
振り返りをまとめて
```

3ヶ月分を評価シートのフォーマットに整形。提出用に出力します。

## Claude Code版との違い

| 項目 | Claude Code | Cursor |
|------|------------|--------|
| 起動方法 | ターミナルで `claude` | Composer (Agent mode) |
| ファイル操作 | 自動で読み書き | エージェントが読み書き |
| CLAUDE.md読み込み | 自動 | 自動 |
| メンターペルソナ | 自動参照 | 自動参照 |
| 日報保存 | sessions/ に自動保存 | sessions/ に自動保存 |

基本的な機能はどちらも同じです。

## トラブルシューティング

**AIがCLAUDE.mdのルールに従わない**
→ Composerのモードが **Agent** になっているか確認。Chatモードではプロジェクトファイルを読みにいかないことがあります。

**メンターの口調が安定しない**
→ Claude系モデルを使ってください。GPT系だとペルソナの再現精度が下がることがあります。

**評価シートが読み込まれない**
→ `evaluation/` フォルダにCSVが入っているか確認。ファイル名に日本語が含まれていても大丈夫です。

**進捗バーが表示されない**
→ ゲーミフィケーション設定が必要です。`gm/gamification.md` を参照してください。

## よくある質問

**Q: Cursor無料版でも使える？**
A: Composer (Agent mode) が使えるプランであれば動作します。AIモデルのリクエスト上限に注意してください。

**Q: Claude Code版と同時に使える？**
A: はい。同じリポジトリを両方で開いても問題ありません。日報やセッション記録は同じ `sessions/` フォルダに保存されます。

**Q: 途中でやめても大丈夫？**
A: いつでも中断・再開できます。戻ってきたときは前回の状態から続きが始められます。
