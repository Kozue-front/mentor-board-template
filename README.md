# GM — Growth Mentoring

AIメンターと一緒に、評価シートを「書かされるもの」から「自分で動かすもの」に変えるワークスペース。

```
═══ Status ═══
CMO Lv.2 マーケ設計者 | XP: 210/400
▓▓▓▓▓▓▓▓▓▓░░░░░ 53%
経営思考 Lv.2 | 意思決定 Lv.1 | 外部発信 Lv.2 | コミュニケーション Lv.2
```

---

## できること

- **評価シートの読み解き** — 6軸の現在レベルと伸びしろを可視化
- **ゴール・テーマ・マイミッションの設定** — AIが構造を整理し、あなたが判断する
- **AIメンターとの壁打ち** — HubSpot共同創業者、グロース戦略家など実在の人物ベースのメンターがフィードバック
- **日報** — 1〜3分で記録。月末に自動集計されて振り返りが書ける
- **ゲーミフィケーション** — XP・レベル・進捗バーで成長を可視化

## 始め方

### 1. クローンする

```bash
git clone https://github.com/Kozue-front/mentor-board-template.git
cd mentor-board-template
```

### 2. 評価シートCSVを置く

`evaluation/` フォルダに自分の評価シートを配置。なければ口頭で伝えてもOK。

### 3. AIツールで開いて話しかける

お使いのツールに合わせたクイックスタートガイドを参照してください:

| ツール | ガイド |
|--------|--------|
| **Claude Code** (CLI) | [gm/quickstart.md](gm/quickstart.md) |
| **Cursor** (エディタ) | [gm/quickstart-cursor.md](gm/quickstart-cursor.md) |

どちらのツールでも、最初のコマンドは同じです:

```
評価シートを見てほしい
```


## よくある質問

**Q: 毎日やらないとダメ？**
→ 最低月1回開けば月次振り返りのデータが溜まります。強制ではないです。

**Q: AIの判断は評価に反映される？**
→ AIは構造の整理だけ。最終的な評価判断は人間（評価者）が行います。

**Q: 途中でやめても大丈夫？**
→ いつでも中断・再開できます。戻ってきたら前回の続きから始められます。

**Q: Claude CodeとCursorどっちがいい？**
→ どちらでも同じ機能が使えます。普段使っているツールでOKです。

## 設計原則

- AIは「答え」を出さない。「構造」を出す。決めるのはあなた
- 選択式の後に必ず理由を一言聞く（思考停止防止）
- メンターのフィードバックは率直。お世辞なし
- 「既存のまま」「スキップ」は常に正当な選択肢
