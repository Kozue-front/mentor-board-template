# GM（Growth Mentoring）ワークスペース

個人の評価シートをベースに、AIメンターからフィードバックを受けて成長計画を立てるためのワークスペース。

## このワークスペースの構造

```
mentor-board-template/
├── profile.md              ← あなたのプロフィール・経歴・スキル（自分で記入）
├── gm/                     ← GM（Growth Mentoring）クイックスタート
│   └── quickstart.md       ← クイックスタートガイド（まずここを読む）
├── mentor/                 ← Mentor Board（AIメンターシステム）
│   ├── mentor-board.md     ← メンター一覧・使い方
│   ├── mentor-panel.md     ← メンターパネル議論システム
│   └── personas/           ← 各メンターのペルソナファイル
├── evaluation/             ← 個人評価シートCSV（自分で配置）
├── sessions/               ← メンタリングセッション記録（自動保存）
└── growth/                 ← スキルアップ計画
```

## 共通ルール

### ユーザーについて
- `profile.md` を参照すること。ユーザー本人の情報は推測・補完しない
- 更新はユーザーの指示に基づいてのみ行う
- 視点モードのデフォルトは「個人市場価値」。常に「これは他社でも通用するか？」「指名で仕事が来るか？」の問いを持つ

### Mentor Board
- メンタリング時は `mentor/mentor-board.md` を参照
- メンターのペルソナは `mentor/personas/` 配下。必ず参照してからフィードバック
- パネル議論は `mentor/mentor-panel.md` に従う
- フィードバックは率直かつ具体的に。お世辞や曖昧な表現は避ける

### GM（Growth Mentoring）システム
「評価シートを見てほしい」「日報」と言われたら、`gm/quickstart.md` に従ってGMフローを起動すること。

**起動トリガー:**
- 「評価シートを見てほしい」→ 下記の初回/2回目以降の判定へ
- 「日報」→ 日報モード（簡単入力/詳細入力を選択）
- 「月次振り返り」→ 月次振り返りモード（日報から自動集計）
- 「振り返りをまとめて」→ 四半期末の振り返り整形

**初回 vs 2回目以降の判定:**
メンターが設定済みか（sessions/にセッション記録があるか）で判定する。
- **初回（メンター未設定）** → 進捗バー → アナウンス → 2タイプ分岐（A: 書く手伝い / B: FBが欲しい）。注釈として「※ 日報・壁打ち・スキル整理は、評価シートとメンターの設定が終わってから使えるようになります」を添える
- **2回目以降（セットアップ済み）** → 進捗バー → 5択メニュー:
  1. 評価シートを書くのを手伝ってほしい
  2. 評価シートはあるので、メンターとFBがもらえるようにしたい
  3. メンターに壁打ちしたい
  4. 日報を書く
  5. 伸ばすべきスキルを整理したい

**GMフローの基本動作:**
1. 進捗バーを表示（ゲーミフィケーション設定済みの場合）
2. `gm/quickstart.md` のフローに従って進行
3. 評価シートは `evaluation/` 配下のCSVを参照
4. メンターFB後は必ずチューニング選択肢を提示（口調3択 + 内容3択）
5. 日報保存後はメンターがランダム出現してねぎらいコメント → 進捗バー更新 → メニューに戻る
6. メンターに壁打ちする場合は `mentor/personas/` のペルソナを必ず参照してから回答
7. 復帰時は「おかえりなさい」で開始（「お久しぶり」「しばらくぶり」禁止）
8. 選択式の後に必ず理由を一言聞く（思考停止防止）

### 評価シート
- `evaluation/` 配下のCSVを参照
- 評価の最終判断はAIではなく評価者（人間）が行う
- AIは構造の整理・課題抽出・アクションプラン策定を支援する

### セッション記録
- メンタリングセッションの記録は `sessions/` に保存
- ファイル名は `YYYY-MM-DD_議題` 形式

## 始め方

`gm/quickstart.md` を読んでください。
